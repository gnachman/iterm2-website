2026-02-26: Revert d651e3a54 to restore BCE behavior. Issue
12723

This reverts commit d651e3a54 ("Use defaults rather than
current SGR color when scrolling lines into view").

The reverted commit broke BCE (Background Color Erase)
compliance, which vim relies on. When vim sets a background
color and scrolls, it expects scrolled-in lines to use the
current SGR background color.  After the reverted commit,
scrolled-in lines used terminal defaults instead, causing
inconsistent background colors in vim.

BCE behavior: when new lines scroll into view, they are
filled with the current SGR background color (via grid's
defaultChar). This is synchronized from the terminal before
each token executes.

Added unit tests for BCE behavior: -
testBCE_ScrollingUsesDefaultCharBackgroundColor -
testBCE_ScrollRectUsesDefaultCharBackgroundColor

Note: This may reintroduce the behavior reported in issue
12512, where scrolled-in lines retain their background color
after SGR 49 reset. However, this is correct BCE behavior -
SGR 49 affects future operations, not cells already written
during scroll.

2026-02-25: Fix status bar hidden behind tab bar in

2026-02-07: [CHERRY PICK TO 3.6.7] Fix paste bracketing race
condition causing out-of-order chunks. Issue 12704

Bug: When pasting multi-line text with bracketed paste mode
enabled, the first chunk (containing ESC[200~ start bracket)
could arrive after subsequent chunks, causing applications
like nano to hang.

Root cause: Commit e0a6352 moved the first chunk write into
the willExpect callback to ensure the expectation was
installed before writing. However, this callback is invoked
via a chain of dispatch_async calls (sync  updateExpectFrom
copy  addExpectation  dispatch_async).  The paste timer
could fire before this chain completed, writing subsequent
chunks directly while the first chunk was still pending.

Fix: Queue the first chunk synchronously before dispatching
sync, using a reference counter (bracketedPastePending) to
also queue subsequent chunks. When willExpect fires,
bracketedPasteDidExpect decrements the counter and flushes
the queue if all blocking conditions are cleared
(_buffering, sendingIsBlocked, bracketedPastePending).

This ensures: - First chunk is in queue before any
subsequent chunks - All writes respect existing queuing
conditions - Correct ordering is preserved when the queue is
flushed
